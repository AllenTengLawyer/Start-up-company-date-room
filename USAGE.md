## 1. 启动应用

```bash
cd startup-assistant
python app.py
```

浏览器访问 `http://localhost:8000`

端口冲突时改用：
```bash
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8002
```

## 2. 创建项目

1. 首次打开显示项目列表页，点击 **新建项目**
2. 填写项目名称（如公司名）
3. 选择本地文件夹作为文档根目录（所有文件都存放在此目录下）
4. 选择项目模式：
   - **早期团队** — 侧重创始人背景核查，文件柜分类较简单
   - **成熟公司** — 完整文件柜 + LDD 尽调功能，自动创建标准分类文件夹
5. 点击创建，系统自动在根目录下生成分类文件夹结构

**切换项目**：点击左侧侧边栏顶部的项目名称，展开项目列表后选择。

## 3. 文件柜：注册与管理文件

### 3.1 扫描并注册文件

1. 进入 **文件柜** 页面
2. 点击 **扫描目录** — 系统扫描根目录下所有文件
3. 扫描结果分两类：
   - **建议分类**：系统根据文件名关键词自动推荐分类
   - **未识别**：无法自动匹配的文件，需手动指定分类
4. 点击 **一键注册建议文件** 批量注册所有有建议分类的文件
5. 对未识别文件，在下拉菜单中手动选择分类后点击注册

> 关键词示例：文件名含"营业执照"→ 自动归入"营业执照与章程"；含"劳动合同"→ 归入"劳动合同"

**重复文件检测**：扫描时若发现同名同大小文件，会弹出提示，可选择跳过或覆盖。

### 3.2 浏览与筛选

- 左侧分类树点击任意分类，右侧只显示该分类下的文件
- 点击分类树顶部"全部"查看所有已注册文件
- 文件列表支持按文件名搜索（顶部搜索框）

### 3.3 批量操作

1. 勾选文件左侧复选框（可多选）
2. 顶部工具栏出现批量操作按钮：
   - **移动分类** — 将选中文件移到指定分类
   - **批量重命名** — 弹出重命名规则设置：
     - 日期前缀（YYYY-MM-DD_）
     - 序号前缀（001_、002_…）
     - 自定义前缀/后缀
   - **删除** — 从文件柜注销（不删除磁盘文件）

### 3.4 版本历史

1. 点击文件行右侧的 **📋** 图标
2. 弹出版本历史面板，显示所有历史快照（时间、大小、哈希）
3. 点击某版本的 **回滚** 按钮，将文件恢复到该版本
4. 点击 **清理旧版本** 可保留最近 N 个版本，删除其余

### 3.5 分类管理

- 左侧分类树右键或点击 **+** 可新增子分类
- 拖拽分类可调整顺序
- 点击分类右侧 **⬇ ZIP** 可将该分类及子分类所有文件打包下载

## 4. 创始人背景核查

### 4.1 添加创始人

1. 进入 **创始人** 页面，点击 **添加创始人**
2. 填写基本信息：姓名、职位、身份证号、入职日期、雇佣类型
3. 保存后进入该创始人的详情页

### 4.2 填写核查清单

核查清单分 6 个维度，共 25 项：

| 维度 | 内容 |
|------|------|
| A. 身份与资质 | 身份证明、学历、职业资格 |
| B. 股权与出资 | 出资证明、股权来源、代持协议 |
| C. 竞业与保密 | 前雇主竞业协议、保密义务 |
| D. 知识产权 | IP 归属协议、发明创造归属 |
| E. 诉讼与合规 | 历史诉讼、行政处罚记录 |
| F. 其他背景 | 犯罪记录、信用记录 |

每项操作：
1. 点击状态下拉框，选择：**已提供 / 部分提供 / 未提供 / 不适用**
2. 在说明框填写补充说明（可选）
3. 点击 **上传文件** 关联该项的证明文件

### 4.3 查看完成度

创始人列表页显示每位创始人的核查完成度评分（百分比）。

## 5. LDD 尽调视图

### 5.1 总览

进入 **LDD** 页面，顶部显示：
- **就绪度评分**（百分比）
- 统计：已提供 / 部分提供 / 未提供 / 不适用 各项数量

LDD 清单分 13 个章节：

| 章节 | 内容 |
|------|------|
| §1 | 集团公司基本文件 |
| §2 | 业务与重大合同 |
| §3 | 借款和担保 |
| §4 | 财务和会计 |
| §5 | 动产和不动产 |
| §6 | 知识产权 |
| §7 | 税务及财政补贴 |
| §8 | 雇员和不竞争 |
| §9 | 保险 |
| §10 | 诉讼、执行及行政处罚 |
| §11 | 网络安全、数据合规 |
| §12 | ESG |
| §13 | 其他 |

### 5.2 更新检查项状态

1. 点击任意检查项展开详情
2. 修改状态：**已提供 / 部分提供 / 未提供 / 不适用**
3. 填写说明（可选）
4. 点击 **+ 添加文件** 关联文件柜中的文件：
   - 可按分类批量导入（选择父分类自动导入其下所有文件）
   - 可逐个选择文件
5. 已关联文件可添加备注说明

### 5.3 TODO 清单

点击顶部 **TODO** 标签页，查看所有高/中风险且状态为"未提供"或"部分提供"的必需项，按风险等级排序。

### 5.4 创始人摘要

LDD 视图底部显示各创始人的核查完成度卡片，快速了解创始人背景核查进度。

## 6. LDD 模板管理

### 6.1 查看内置模板

点击 LDD 视图顶部 **模板管理** 按钮：
- 内置模板：天使轮、A 轮、B 轮（不可修改/删除）
- 每个模板显示包含的检查项数量和轮次类型

### 6.2 应用模板到项目

1. 选择目标模板
2. 点击 **应用到当前项目**
3. 确认后，当前项目的 LDD 清单将被替换为模板内容（原有状态数据清空）

### 6.3 创建自定义模板

1. 点击 **新建模板**
2. 填写模板名称、轮次类型、描述
3. 逐项添加检查项（章节号、条目号、标题、风险等级）
4. 保存后可应用到任意项目

### 6.4 导出模板

点击模板右侧 **导出 JSON**，下载模板文件，可分享给其他用户导入使用。

## 7. 导出功能

进入 **导出** 页面，提供以下导出选项：

### HTML 报告
- 点击 **导出 HTML（中文）** 或 **导出 HTML（英文）**
- 浏览器直接打开，可用浏览器"打印"功能另存为 PDF

### Word 报告 (.docx)
- 点击 **导出 Word（中文）** 或 **导出 Word（英文）**
- 包含：评分摘要表 + 各章节检查项明细（状态、关联文件、说明）

### LDD 文件包 (ZIP)
- 点击 **导出 LDD ZIP**
- ZIP 内按 `§章节_章节名/条目号_条目名/` 结构组织文件
- 包含 `LDD_Notes.txt`：所有条目的说明和文件清单

### 文件夹复制
- 填写目标目录路径，点击 **复制到文件夹**
- 将文件柜所有已注册文件按分类树结构复制到目标目录
- 返回复制数量和跳过文件列表

### JSON 备份
- 点击 **下载 JSON 备份**
- 备份内容：项目信息、分类树、文件记录、LDD 状态与映射、创始人数据

### JSON 导入（恢复）
- 点击 **导入 JSON 备份**，选择备份文件
- 系统创建新项目（名称后缀"(导入)"），不覆盖现有项目

### 创始人报告
- 点击 **导出创始人报告**
- 下载 ZIP，包含：
  - `创始人背景核查报告.docx`（所有创始人的核查清单汇总）
  - 各创始人文件夹（含上传的所有附件）

## 8. 全文搜索

1. 点击顶部导航栏 **🔍 搜索** 按钮
2. 输入关键词，按 Enter 或点击搜索
3. 结果显示：文件名、所属分类、匹配文本片段（高亮）
4. 点击结果可跳转到文件柜对应文件
5. 按 **ESC** 或点击返回退出搜索

> 全文搜索支持 PDF、Word (.docx)、TXT 文件内容。首次注册文件时自动建立索引，后续搜索即时响应。

## 9. 常见问题

**Q: 启动后浏览器显示"无法连接"？**
A: 确认 `python app.py` 已在终端运行且无报错。默认端口 8000，若被占用请改用 8002。

**Q: 扫描目录后没有建议分类？**
A: 系统根据文件名关键词匹配，确保文件名包含可识别的关键词（如"营业执照"、"劳动合同"等）。无法识别的文件需手动选择分类。

**Q: Word 导出中文乱码？**
A: Word 报告使用 Microsoft YaHei 字体，需在 Windows 系统上打开效果最佳。macOS 可安装该字体或使用 HTML 报告替代。

**Q: 文件夹复制提示"路径超过系统长度限制"？**
A: Windows 默认路径长度限制为 260 字符。请选择路径较短的目标目录，或在系统设置中启用长路径支持。

**Q: 如何迁移到新电脑？**
A: 导出 JSON 备份，将整个 `startup-assistant` 文件夹（含数据库）复制到新电脑，重新安装依赖后启动即可。或在新电脑上导入 JSON 备份恢复项目数据。

**Q: LDD 评分如何计算？**
A: 评分 = (已提供 + 部分提供 × 0.5) ÷ (必需项总数 - 不适用项数) × 100%

**Q: 应用模板后原来的 LDD 状态会丢失吗？**
A: 是的，应用模板会替换当前项目的 LDD 检查项列表，原有状态数据会被清空。建议先导出 JSON 备份再操作。
